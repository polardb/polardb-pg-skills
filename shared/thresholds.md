# 通用阈值定义

所有技能共享的阈值参考值。

## 事务和 Vacuum

| 指标 | 正常 | 警告 | 危险 | 说明 |
|------|------|------|------|------|
| 事务ID年龄 | < 1.5亿 | 1.5-20亿 | > 20亿 | 接近 20 亿会触发保护性冻结 |
| 死元组比例 | < 10% | 10-30% | > 30% | 需要执行 VACUUM |
| VACUUM 间隔 | < 1天 | 1-7天 | > 7天 | 大表应更频繁 |

## 索引

| 指标 | 正常 | 警告 | 危险 | 说明 |
|------|------|------|------|------|
| 索引扫描次数 | > 1000 | 100-1000 | 0 | 0 表示未使用 |
| 索引大小/表大小 | < 30% | 30-50% | > 50% | 索引过大需优化 |

## 连接

| 指标 | 正常 | 警告 | 危险 | 说明 |
|------|------|------|------|------|
| 空闲事务时长 | < 60秒 | 60-300秒 | > 300秒 | 可能持有锁 |
| 连接数占比 | < 50% | 50-80% | > 80% | 占 max_connections 比例 |
| 长查询时长 | < 5分钟 | 5-30分钟 | > 30分钟 | |

## 复制

| 指标 | 正常 | 警告 | 危险 | 说明 |
|------|------|------|------|------|
| 复制延迟 | < 1MB | 1-100MB | > 100MB | 主从数据差异 |
| WAL 堆积 | < 1GB | 1-10GB | > 10GB | 未消费的 WAL |

## 性能

| 指标 | 正常 | 警告 | 危险 | 说明 |
|------|------|------|------|------|
| 缓存命中率 | > 99% | 95-99% | < 95% | shared_buffers 效率 |
| 慢查询阈值 | < 100ms | 100ms-1s | > 1s | 单次查询执行时间 |
| 锁等待时间 | < 100ms | 100ms-1s | > 1s | |

## 存储

| 指标 | 正常 | 警告 | 危险 | 说明 |
|------|------|------|------|------|
| 磁盘使用率 | < 70% | 70-90% | > 90% | |
| 表膨胀率 | < 20% | 20-50% | > 50% | 需要 VACUUM FULL 或 pg_repack |

## 配置参数建议值

| 参数 | 建议值 | 说明 |
|------|--------|------|
| shared_buffers | 内存的 25% | 最大不超过 8GB (Windows) 或 16GB (Linux) |
| effective_cache_size | 内存的 75% | 帮助优化器选择执行计划 |
| work_mem | 4-64MB | 谨慎设置，会被并发连接放大 |
| maintenance_work_mem | 256MB-1GB | 用于 VACUUM, CREATE INDEX |
| random_page_cost | SSD: 1.1-2, HDD: 4 | 磁盘类型决定 |
| effective_io_concurrency | SSD: 200, HDD: 2 | |
| checkpoint_completion_target | 0.8-0.9 | 分散 checkpoint 压力 |
